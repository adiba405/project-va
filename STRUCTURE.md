## 🧭 Simple Architecture Overview

This project is split into two clear areas:

- **Backend**: Flask app and API logic in `backend/`.
- **Frontend**: Static pages, UI behavior, and API calls in `frontend/`.

A browser page loads HTML and JavaScript from the frontend, then talks to the backend APIs to fetch and update notes, tasks, files, and AI content.

## 🏗️ Architecture Overview

### Backend (Flask MVC)

```
Request → Routes (Blueprint) → Controllers (Logic) → Database (MongoDB)
Response ← Routes (Blueprint) ← Controllers (Logic) ← Database (MongoDB)
```

**Routes**: Define API endpoints and HTTP methods
**Controllers**: Contain business logic and database operations
**Models**: Schema definitions (currently in MongoDB, expandable)
**Utils**: Shared utility functions (password hashing, response formatting)

### Frontend (Vanilla JavaScript)

```
HTML (Structure) → CSS (Styling) → JavaScript (Behavior)
```

- **Static Pages**: HTML files for each feature
- **Central API Client**: app.js handles all API communication
- **Styling System**: Pastel theme with dark mode support

## 📦 API Organization

### By Feature (RESTful)

```
/api/auth/        - User authentication
  POST /register  - Create new user
  POST /login     - Login & get token
  GET  /profile   - Get user profile

/api/notes/       - Note management
  GET  /?page=<n>&limit=<m>  - List paginated notes
  POST /         - Create note
  GET  /<id>     - Get specific note
  PUT  /<id>     - Update note
  DELETE /<id>   - Delete note
  POST /<id>/share  - Share note with another user
  DELETE /<id>/share/<share_user_id> - Revoke share
  GET  /shared    - List notes shared with current user
  GET  /<id>/shares - List users shared with

/api/files/       - Uploaded files
  GET  /         - List uploaded files
  POST /         - Upload a file
  DELETE /<id>   - Delete file
  GET  /<id>/download - Download file

/api/tasks/       - Task management
  GET  /?page=<n>&limit=<m>  - List paginated tasks
  POST /         - Create task
  PUT  /<id>     - Update task
  DELETE /<id>   - Delete task

/api/ai/          - AI features
  POST /ask       - Answer question
  POST /summarize - Summarize text or uploaded file
  POST /quiz      - Generate quiz

/api/user/        - User management
  GET  /profile   - Get profile
  PUT  /profile   - Update profile
  POST /change-password - Change password
  GET  /stats     - Get statistics
```

## 🗄️ Database Schema

### MongoDB Collections

**users**
```javascript
{
  _id: ObjectId,
  name: String,
  email: String (unique),
  password: String (bcrypt hashed),
  created_at: DateTime
}
```

**notes**
```javascript
{
  _id: ObjectId,
  user_id: String,
  subject: String,
  topic: String,
  content: String,
  drawing: String (base64 dataURL),
  tags: [String],
  created_at: DateTime,
  updated_at: DateTime
}
```

**tasks**
```javascript
{
  _id: ObjectId,
  user_id: String,
  title: String,
  deadline: DateTime,
  priority: String (low|medium|high),
  status: String (pending|completed),
  created_at: DateTime
}
```

## 🚀 Development Workflow

1. **Start Backend**
   ```bash
   .\venv\Scripts\python.exe backend/app.py
   # Runs on http://localhost:5000
   ```

2. **Open Frontend**
   - Navigate to `http://localhost:5000/` in browser
   - Or open `frontend/index.html` directly

3. **API Testing**
   - Use browser DevTools → Network tab
   - Or use Postman/Insomnia
   - Include JWT token in Authorization header: `Bearer <token>`

## 📋 Best Practices

✅ **Controllers** - Keep business logic separate from routes
✅ **Routes** - Define endpoints clearly, delegate to controllers
✅ **Utils** - Extract reusable functions for sharing
✅ **Frontend** - Centralized API client (app.js) for consistency
✅ **Database** - User isolation (users only see their own data)
✅ **Security** - Passwords hashed, JWTs for auth, CORS configured

## 📚 How to Add New Features

### Backend Feature
1. Create controller in `controllers/<feature>_controller.py`
2. Create route in `routes/<feature>_routes.py`
3. Register blueprint in `app.py`

### Frontend Feature
1. Create HTML page `frontend/<feature>.html`
2. Add links in navbar across all pages
3. Use `apiFetch()` from app.js for API calls

## 🔒 Security Checklist

- [ ] CORS configured for allowed origins
- [ ] Environment variables secured (.env not in git)
